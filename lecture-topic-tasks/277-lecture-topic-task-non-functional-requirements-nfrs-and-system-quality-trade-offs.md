## ğŸ“ Description
This lecture topic task explores how NFRs shape the architecture and behavior of a system. While functional requirements specify what the software does, NFRs determine how well it performs. Students will identify key NFR categories, describe measurable quality attributes, and analyze how improving one may negatively affect another. Examples include conflicts like performance vs. security or usability vs. reliability. The task connects concepts from the Requirements Engineering and Software Design lectures, using real-world cases such as an airline timetable or e-commerce platform to demonstrate how engineers balance system qualities under constraints.
